# S8.Phase1.15: Matter-stage explainer library

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

Lawyers answer the same questions over and over. "What happens at discovery?" "What to expect during a real estate closing?" Phase 1 ships an operator-curated explainer library tagged by `practice_area` and `matter_stage`. The lawyer assigns one or more articles to a matter; the client sees them under "Understanding your case." Generic FAQ is skipped; the explainers are matter-stage-aware and Ontario-specific.

## Acceptance criteria

- [ ] New table `explainer_articles` (id, slug UNIQUE, title, body_html, practice_area, matter_stage, ordering, published, created_at, updated_at).
- [ ] New table `matter_explainer_assignments` (id, matter_id, article_id, assigned_by_lawyer_id, assigned_at).
- [ ] Operator-authored content seeded via migration. Phase 1 seed list: 10 articles covering the most common Ontario practice-area stages (operator authors after this story lands; the migration scaffolds the table and inserts a placeholder seed list with slugs only: actual body content is added in a separate operator content pass, not by @dev).
- [ ] New API route `GET /api/portal/[firmId]/matters/[matterId]/explainers`: returns the list of articles assigned to the matter. Both lawyer and client sessions can read.
- [ ] New API route `POST /api/portal/[firmId]/matters/[matterId]/explainers` body `{ article_id }`. Lawyer (admin or staff) only. Assigns an article to the matter.
- [ ] New API route `DELETE /api/portal/[firmId]/matters/[matterId]/explainers/[assignment_id]`. Removes the assignment.
- [ ] Lawyer-side surface: on `/portal/[firmId]/clients/[matterId]`, a panel labeled "Explainers" shows currently-assigned articles and a "+ Assign" button opening a picker filtered by `(matter.practice_area, matter.matter_stage)`.
- [ ] Client-side surface: `/portal/[firmId]/m/[matterId]/explainers` renders the assigned articles as a read-only list with each article expandable. Sticky table-of-contents on desktop.
- [ ] Every article body clears LSO Rule 4.2-1: no outcome promises, no specialist or expert language, no unverifiable superlatives. The migration scaffolds the slug list and the article-author runbook documents the rule.
- [ ] Per-firm authoring is NOT in Phase 1. The articles table is operator-write-only via direct database access; firms cannot author or edit.
