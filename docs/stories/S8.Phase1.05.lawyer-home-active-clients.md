# S8.Phase1.05: Lawyer home: triage on top, active clients below

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

The lawyer logs in and wants two things in this order: the inbound triage queue (decisions that must be made today) and the active client list (matters that need attention). Today the home redirects to the dashboard tab. Phase 1 reshapes the home into a two-section page: the existing Triage Portal on top, the new active-clients surface below.

## Acceptance criteria

- [ ] `/portal/[firmId]/page.tsx` no longer redirects to dashboard. It renders a server component that composes the triage queue (existing) and the active-clients list (new).
- [ ] Active-clients section shows matters with `matter_stage IN ('retainer_pending', 'active', 'closing')` sorted by `updated_at DESC`.
- [ ] Each card surfaces: client name, matter type, current stage badge, last activity timestamp, count of pending items (unread client messages, pending stage-events the lawyer must act on).
- [ ] Filter chip: "needs my attention": toggles the list to matters with `pending_items > 0`.
- [ ] Role gating: admins see all matters; staff see only matters where they are the `lead_id` or in `assignee_ids` (from Story 2's `canSeeMatter`).
- [ ] New API route `GET /api/portal/[firmId]/active-clients` returns the list, accepts `?filter=needs-attention`.
- [ ] Existing `/portal/[firmId]/dashboard` page stays as is, accessible via the tab nav. The home is no longer a redirect to dashboard; the tab nav still surfaces the dashboard link.
- [ ] Existing bookmarks to `/portal/[firmId]` continue to work (the page renders content rather than redirecting).
