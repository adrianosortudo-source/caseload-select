# S8.Phase1.16: Minimal per-client iframe embed

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

A firm wants to surface a third-party dashboard (Airtable view, Looker board, Notion page) inside the client portal so the client experiences the firm's bespoke surface without leaving the portal. Phase 1 ships a minimal sandboxed embed slot per matter. The firm provides a pre-authenticated URL (public, or with its own auth token in the path). The portal renders it in an iframe with sandbox attributes plus a per-firm CSP allow-list. No auth bridge in Phase 1.

## Acceptance criteria

- [ ] New column `client_matters.embed_url TEXT` (in Story 3's migration).
- [ ] New column `intake_firms.embed_origins JSONB DEFAULT '[]'::jsonb` (in Story 13's migration). One or more allowed origins per firm.
- [ ] New page route `/portal/[firmId]/m/[matterId]/embed/page.tsx`: server component. Reads the matter's `embed_url`, validates the URL's origin against `intake_firms.embed_origins`, renders an iframe with `sandbox="allow-scripts allow-same-origin allow-forms"` and a route-level `Content-Security-Policy` response header that allow-lists the origin.
- [ ] If `embed_url` is null or the origin is not in the allow-list, the page renders a placeholder "No embed configured for this matter." No 500 errors.
- [ ] Module-shortcut row on the client matter-stage home (Story 4) surfaces the Embed card only when `embed_url` is set.
- [ ] New API route `PUT /api/portal/[firmId]/matters/[matterId]/embed-url`: operator role only. Sets the URL on a matter. No firm-facing UI.
- [ ] No session identity is passed through the iframe. The portal does NOT post an auth message to the embedded app via postMessage. Auth bridge is explicitly out of Phase 1 scope.
- [ ] The CSP includes `frame-src` listing the allowed origins; everything else is blocked.
