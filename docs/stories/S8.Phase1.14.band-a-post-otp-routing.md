# S8.Phase1.14: Band A post-OTP routing pipeline

## User narrative

The lawyer presses Take on a Band A screened lead. A chain of side effects fires: the GHL `taken` webhook (existing), a new `client_matters` row created at stage `intake`, the welcome draft generated, default lead and default assignees snapshotted, the welcome-draft banner surfaced on the lawyer's client detail page. This story wires the chain end-to-end and makes the take flow observable through events.

## Acceptance criteria

- [ ] Take handler at `/api/portal/[firmId]/triage/[leadId]/take/route.ts` is the single composition point for the Band A side effects: GHL webhook (existing), matter creation (Story 3), routing snapshot (Story 13), welcome draft (Story 8).
- [ ] The handler runs the side effects in this order: (a) flip `screened_leads.status='taken'` (existing), (b) insert `client_matters` row with stage `intake` and snapshotted routing config, (c) call `buildWelcomeDraft` and write the draft onto the matter, (d) enqueue an internal-thread notification to the lead lawyer ("New client matter ready for review"), (e) fire the GHL `taken` webhook with the new `matter_id` field.
- [ ] If any side effect fails, the take is still committed (the lead is taken) but the failure surfaces in the operator console (`/admin/take-failures` minimal page). Operator runbook covers manual recovery.
- [ ] GHL webhook contract bumped from v2 to v3, adding `matter_id` to the common envelope and a new `client_matter_created` extension on the `taken` action payload. Contract update lives at `docs/ghl-webhook-contract.md`.
- [ ] Take response includes `{ matter_id, welcome_draft_required: true }` so the lawyer-side UI can redirect from the brief page to the client detail page after take.
- [ ] Non-Band-A takes (Band B, C, D) do NOT trigger matter creation. The existing take flow continues unchanged for them; the response omits `matter_id`.
- [ ] Take is idempotent on `lead_id`: a second take attempt returns 409 with the existing `matter_id` if a matter was already created.
