# S8.Phase1.13: Default-lead and default-assignees firm config

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

When a Band A lead lands and the lawyer takes it, the system needs to know who the matter belongs to and who else should see it. For a solo firm, the lawyer is always the lead. For a 2-lawyer firm split by practice area (one corporate lawyer and one family-law lawyer), the lead is routed by practice area. Phase 1 introduces a small firm-config surface stored as JSONB on `intake_firms`.

## Acceptance criteria

- [ ] Migration adds the following columns on `intake_firms`:
  - `default_lead_by_practice_area JSONB DEFAULT '{}'::jsonb`: map of `{ practice_area: firm_lawyer_id }`.
  - `default_lead_id UUID REFERENCES firm_lawyers(id) ON DELETE SET NULL`: fallback when no practice-area match.
  - `default_assignees JSONB DEFAULT '[]'::jsonb`: array of `firm_lawyer_id` UUIDs, snapshotted onto each new matter at take time.
  - `client_files_locked BOOLEAN DEFAULT false` (Story 10 consumer).
  - `subdomain TEXT UNIQUE` (Story 12 consumer).
  - `embed_origins JSONB DEFAULT '[]'::jsonb`: CSP allow-list of origins for matter embeds (Story 16 consumer).
- [ ] Pure resolver `resolveDefaultLead(firmConfig, practiceArea): { lead_id: string | null }`. Returns the practice-area match if present, otherwise the `default_lead_id`, otherwise null.
- [ ] Take-action handler (Story 3) calls `resolveDefaultLead` and `firmConfig.default_assignees` to populate the new matter's `lead_id` and `assignee_ids` columns.
- [ ] When the resolver returns null lead_id, the take handler returns 422 with a clear error message: "Firm routing config missing: set default_lead_id or default_lead_by_practice_area on intake_firms before taking a Band A lead." Operator-actionable.
- [ ] No firm-facing settings UI in Phase 1. The operator sets these columns directly during onboarding via the operator console.
