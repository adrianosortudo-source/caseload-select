# S8.Phase1.12: Branded subdomain per firm

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

The client lands on the portal from a magic-link invite email. The URL in the link should feel like the firm's, not like a third-party SaaS. Phase 1 extends the existing S9 custom-domain machinery to support a per-firm branded subdomain (e.g., `portal.powellfirm.ca`) without requiring the firm to set up a full custom domain. The operator picks the subdomain during onboarding and configures the DNS.

## Acceptance criteria

- [ ] New column `intake_firms.subdomain TEXT UNIQUE` (declared in the routing-config migration, Story 13). Distinct from `custom_domain` which is a full apex domain.
- [ ] Existing edge middleware (`src/middleware.ts`) extended: hostname detection now matches against both `custom_domain` and `subdomain`. A request to `portal.powellfirm.ca` rewrites to `/portal/[firmId]/...` with the firmId resolved from `subdomain`.
- [ ] Subdomain values are validated server-side: lowercase, hyphen-allowed, no dots, 3-63 chars, not in a reserved-words list (`app`, `admin`, `api`, `www`, `mail`, `portal-global`).
- [ ] Magic-link invite URLs (Story 1) prefer the firm's `custom_domain` if set, otherwise `subdomain` if set, otherwise fall back to `app.caseloadselect.ca`.
- [ ] No subdomain self-service. The operator updates `intake_firms.subdomain` directly during firm onboarding. The Vercel DNS configuration is the operator's responsibility per the existing S9 runbook.
- [ ] The lawyer portal continues to be served at the firm's subdomain (or custom domain). The legacy admin paths (`/admin/*`) only resolve at `app.caseloadselect.ca`; a request to `portal.powellfirm.ca/admin/triage` returns 404 to prevent operator console leakage.
