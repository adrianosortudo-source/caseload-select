# S8.Phase1.09: 5-minute email notification batching

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

A chatty client sends twenty messages in a five-minute window. Without batching, the lawyer's inbox is hit with twenty notification emails. The lawyer turns off notifications. The portal dies. Phase 1 routes all matter-event notifications into a queue table, drains it every five minutes, and produces one digest email per recipient per window. Transactional emails (magic-link invite, welcome send, password reset) bypass the queue.

## Acceptance criteria

- [ ] New table `notification_outbox` (id, recipient_user_id, recipient_email, firm_id, matter_id, event_type, event_payload JSONB, status CHECK IN ('queued', 'sent', 'failed'), batch_id UUID, created_at, sent_at).
- [ ] All event-driven notifications (new message in either channel_type, new file upload, stage transition, lawyer-assigned explainer) write to `notification_outbox` instead of firing Resend directly.
- [ ] New cron route `GET /api/cron/notification-batch` (Bearer auth, same `PG_CRON_TOKEN`/`CRON_SECRET` pattern as existing crons). Drains queued rows older than 5 minutes, groups by recipient_email, sends one digest email per group, marks rows `sent`.
- [ ] pg_cron schedule: `notification-batch-5m` at `*/5 * * * *`, calling `cron_internal.call_cron_route('/api/cron/notification-batch')`. Migration `20260520_s8p1_notification_batch_cron.sql`.
- [ ] Digest email template surfaces: section per matter with N events listed under each. Single CTA: "Open the portal."
- [ ] Per-recipient setting `firm_lawyers.email_notifications_enabled BOOLEAN DEFAULT true`: when false, queued rows for that recipient are dropped at drain time (no per-event preferences, no per-channel preferences; one toggle).
- [ ] Transactional emails (invite, welcome, magic link) continue to call `sendEmail` directly. They do NOT pass through the outbox.
- [ ] Failed sends increment `attempts` and retry the next drain. After 5 failed attempts, the row is marked `failed` and surfaces in `/admin/notification-outbox` (operator-only view, basic listing).
