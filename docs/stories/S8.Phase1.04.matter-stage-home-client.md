# S8.Phase1.04: Matter-stage home for the client

**Status:** Shipped (2026-05-22)
**Assignee:** @dev
**Epic:** S8.Phase1
**Created:** 2026-05-18

## User narrative

A client logging into the portal should see one thing first: what they need to do next. Not a message inbox, not a list of documents, not a generic "welcome." The hero element on the matter-stage home is a single next-action CTA driven by `matter_stage`. Below it sits a timeline of recent activity, and below that a row of module shortcuts (messages, files, explainers, embed).

## Acceptance criteria

- [ ] New page route `/portal/[firmId]/m/[matterId]/page.tsx` server-component, gated by `getMatterSession()` (Story 1).
- [ ] Hero block renders one of five variants keyed by `matter_stage`:
  - `intake`: "Your firm is preparing your matter. The lead lawyer will be in touch shortly." No CTA, status only.
  - `retainer_pending`: "Sign the retainer to begin work." CTA: "Open the retainer email" (just a hint; the lawyer owns the document workflow per DR-032). No retainer signing inside the portal.
  - `active`: "Latest update from the firm: {last_client_message.preview}." CTA: "Open the thread."
  - `closing`: "Your matter is wrapping up. The firm will share final documents shortly." No CTA.
  - `closed`: "Matter closed on {closed_at}." CTA: "Leave a review for the firm" (links to the firm's review URL from `intake_firms.branding.review_url` if set).
- [ ] Timeline panel below the hero, reverse-chronological list of `matter_stage_events` + recent `matter_messages` (combined feed, max 10 items).
- [ ] Module shortcut row at the bottom: Messages, Files, Explainers (if assigned), Embed (if `embed_url` set on the matter). Each is a card linking to the corresponding sub-route.
- [ ] Firm branding (logo, primary color) pulled from `intake_firms.branding`. No CaseLoad Select branding on the client-facing surface.
- [ ] New API route `GET /api/portal/[firmId]/matters/[matterId]/home` returns the JSON payload the page renders. Client sessions only.
- [ ] No outcome promises, no specialist or expert language, no unverifiable superlatives. Every copy snippet clears LSO Rule 4.2-1.
